pg-Buffer Manager的ring buffer

2024-06-11
  • pg 一般情况使用的是buffer pool进行缓存的。
  • 假设要scan一个超级大表，并且只使用一次，使用buffer pool并不合适，因为缓存里的数据会被淘汰掉，导致其他请求并不能利用缓存加速。
  • 这时候应该使用额外的一种缓存，即：ring buffer，它在使用完成后会被立即回收掉。

哪些情况使用ring buffer

  • scan的数据超过buffer pool大小的1/4。
  • 一些特定的sql
    • copy from
    • create table as
    • create/refresh meterialized view
    • alter table
  • vaccum 进程执行的时候也会用到ring buffer
