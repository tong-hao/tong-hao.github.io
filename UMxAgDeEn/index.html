B+ 对比 LSM

2024-06-01

图操作的典型场景

  • KHop， 从一个顶点出发，扩展N层后返回结果。
  • 获取一个点的边
  • 点或边的属性过滤
  • 基于时间过滤
  • 写的同时有大量的读

B+ VS LSM

  • B+ 排序好的数据
  • LSM trees append updates
  • B+ trees have better sequential read/write performance
  • LSM trees have advantages in random write data
  • LSM use WAL to perform updates
  • LSM trees using background merging makes the performance fluctuation difficult to predict
  • TuGraph chooses B+ trees
